# Step 2: 信息收集 + 初版设计

## 2.1 派出3个Explore subagent (并行)

**Agent 1: 项目 Bird Eye View**
```
任务: 获取项目整体结构和架构概览
输出:
- 目录结构树
- 核心模块关系图
- 主要入口点
- 技术栈概览
```

**Agent 2: 功能相关 Bird Eye View (Global + Local)**
```
任务: 基于Step 1的需求，探索相关代码区域
输出:
- 相关文件列表
- 现有类似功能的实现方式
- 可复用的模式/工具
- 需要修改的文件预估
```

**Agent 3: 功能局部 View (如存在)**
```
任务: 如果项目中已有类似功能，深入分析其实现
输出:
- 具体实现细节
- 接口设计
- 依赖关系
- 可借鉴的代码片段
```

## 2.2 Dump探索报告

将3个agent的输出汇总到 `step2-exploration.md`

## 2.3 初版设计

基于探索结果，进行初版设计，包含：

**A. 流程图 (ASCII)**
```
+─────+    +─────+    +─────+
|  A  |--->|  B  |--->|  C  |
+─────+    +─────+    +─────+
```

**B. 参数列表**

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| param1 | type | default | desc |

**C. 结构描述**
- 模块划分
- 类/函数设计
- 数据流

**D. 数学方程 (如适用)**

$$
\text{公式}: y = f(x)
$$

**人类可读解释**: 这个公式表示...

## 2.4 用户确认循环

展示初版设计，询问用户：
- 这是你想要的吗？
- 有什么需要修改？

重复直到收敛。

## 2.5 记录

更新 `step2-exploration.md` 包含最终设计
